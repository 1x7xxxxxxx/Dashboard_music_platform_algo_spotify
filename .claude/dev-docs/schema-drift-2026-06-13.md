## Durable rule
**Every schema change ships as `migrations/0NN.sql` (+ the matching `*_schema.py` /
`init_db.sql`) ‚Äî never a manual `ALTER` on prod.** Then prod == init_db.sql +
migrations by construction. Run `make schema-check PROD_SSH=user@host` after any
prod schema touch (and consider a nightly cron on the server) to catch regressions.
Error class: `prod-canonical-schema-drift` (`error-classes.md`).
