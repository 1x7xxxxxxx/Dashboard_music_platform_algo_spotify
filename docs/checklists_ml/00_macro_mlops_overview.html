Phase 1 — Data Preparation & EDA

Phase 1 — Data Preparation & EDA

Initialization

Set up the working environment. - Import core libraries: pandas, numpy, scikit-learn, seaborn, matplotlib - Read source file (CSV, XLSX, SQL, time-series store) - Pin version of every dependency in pyproject.toml / requirements.txt

Phase 1 — Data Preparation & EDA

Initial audit

Verify dataset structural integrity. - df.head(), df.tail() — first / last rows - df.dtypes — type sanity check (int / float / object / datetime) - Detect duplicates: df.duplicated().sum() - Detect inputs with irregular relations between two variables (e.g. listeners > streams)

Phase 1 — Data Preparation & EDA

Column cleanup

Make variables easy to call from code. - Lowercase: .str.lower() - Strip whitespace - Replace spaces with underscores (snake_case) - Drop free-text columns that won't be used downstream

Phase 1 — Data Preparation & EDA

Descriptive analysis

Understand statistical distribution and quality. - df.describe() — mean, std, min/max - ydata-profiling (ex Pandas Profiling) for a complete audit report - great-expectations for schema validation + profiling + documentation - Identify outliers and zero-as-missing values

Phase 1 — Data Preparation & EDA

Spoilers / leakage hunt

Avoid data leakage — predicting from the answer. - Drop columns that reveal the future (e.g. "death_date" when predicting survival, or "amount_repaid" when predicting default) - Drop columns derived from the target (multicollinearity with target)

Phase 1 — Data Preparation & EDA

Missing data analysis

Identify the right imputation strategy. - Compute % NaN per column - Visualize: missingness matrix (Missingno), missingness dendrogram (do features go missing together?)

Phase 1 — Data Preparation & EDA

Visualization (Data Viz)

Detect relations and visual patterns. - Numerical relations: pairplot, jointplot, heatmap (Pearson correlation) - Distributions: histograms, joint plot, box plot, violin plot, dependence matrix, RFE (recursive feature elimination) - Check power-law distribution if applicable - Non-parametric tests when normality fails

Phase 1 — Data Preparation & EDA

Phase 1 deliverables

  • Cleaned dataset with documented schema
  • ydata-profiling HTML report attached to project repo
  • List of dropped columns + reason
  • Missingness strategy documented in plan.md

Phase 2 — Feature Engineering & Preprocessing

Phase 2 — Feature Engineering & Preprocessing

Train / test split

Guarantee an honest evaluation. - train_test_split 80/20 or 70/30 for IID data - Time series: TimeSeriesSplit — chronological split, never random shuffle - Verify both classes appear in both train and test

Phase 2 — Feature Engineering & Preprocessing

Imputation

Fill the gaps without biasing the model. - Compute statistics on TRAIN only, apply to TEST - Numerical: mean / median (SimpleImputer) - Categorical: mode / constant - Advanced: KNNImputer, IterativeImputer - Volume variables: log1p transform; clip negative artifacts to 0

Phase 2 — Feature Engineering & Preprocessing

Categorical encoding

Transform text into numbers. - One-Hot (get_dummies, OneHotEncoder) — use drop_first=True to avoid multicollinearity - Label Encoding — for ordinal categories or high cardinality - Frequency Encoding — replace by frequency of occurrence - Done BEFORE SMOTE — SMOTE needs numeric distances

Phase 2 — Feature Engineering & Preprocessing

Standardization / scaling

Same scale across all variables. - StandardScaler — mean=0, std=1 (ideal for SVM, KNN, Linear) - MinMaxScaler — [0, 1] (useful for neural networks) - Always: fit on TRAIN, transform on TEST — never the other way around

Phase 2 — Feature Engineering & Preprocessing

Feature selection

Reduce noise and complexity. - Variance Inflation Factor (VIF < 5) - Log-transform (np.log1p) before scaling on extreme volume variables - Statistical tests: chi-squared, Kolmogorov-Smirnov, Spearman univariate - Wrappers: subset selection - Embedded: Lasso (automatic via L1 penalty) - Binning to bucket continuous values into intervals - Temporal dynamics: 7-day growth vs 21-day moving average → detect organic vs artificial signal - Cross-metrics: ratios like Listeners/Streams as retention scores - Drop zero-variance columns - Drop features with leakage (correlation with target via spurious mechanism) - Recursive Feature Elimination (RFE) - Radviz, parallel coordinates plot

Phase 2 — Feature Engineering & Preprocessing

Target definition

Define the y you actually want to predict. - Log-Knee (DW) method + active median for thresholding - Verify NO feature is computed from the target - Binarization: turn an asymmetric continuous target (with many zeros) into a binary problem (Trigger: yes/no)

Phase 2 — Feature Engineering & Preprocessing

Class balancing

Help the model see rare cases. - Apply ONLY on TRAIN set — never on val / test - Oversampling: SMOTE, ADASYN (synthetic data generation) - Undersampling: random removal of majority class - SMOTE rationale: forces XGBoost to study successes — prevents lazy "always predict 0" behavior, preserves test-set distribution

Phase 2 — Feature Engineering & Preprocessing

Phase 2 deliverables

  • Train / test split saved as artifacts (joblib)
  • Preprocessing Pipeline saved (scaler + imputer + encoder)
  • Feature catalog with formula + source

Phase 3 — Modeling & Optimization

Phase 3 — Modeling & Optimization

Baseline model

Establish a floor score. - DummyClassifier (most_frequent / stratified) or DummyRegressor (mean / median) - If your real model doesn't beat it by ≥ 5 %, treat as red flag

Phase 3 — Modeling & Optimization

Algorithm selection

Test different approaches. - Classification: LogReg, NB, SVM, KNN, DT, RF, XGBoost, LightGBM, CatBoost, TPOT - Regression: LinearReg, SVR, KNN, DT, XGBoost, LightGBM, DummyRegressor - Time series: ARIMA, LSTM, RNN, TCN - AutoML: TPOT (genetic algorithms), AutoGluon, FLAML - Stacking — careful with time series (no future leakage in meta-model)

Phase 3 — Modeling & Optimization

Cross-validation

Verify model stability. - cross_val_score with k-fold - Confirms the model performs well across different parts of training data - Time series: TimeSeriesSplit — train always BEFORE val - Both classes must appear in train AND test of every fold

Phase 3 — Modeling & Optimization

Hyperparameter optimization

Find the optimal tuning. - GridSearchCV — exhaustive but slow - RandomizedSearchCV — random sampling, faster - Bayesian: Optuna, Hyperopt - Early stopping on boosted trees: early_stopping_rounds — algorithm stops just before memorizing training data

Phase 3 — Modeling & Optimization

Overfit / underfit diagnosis

Check whether the model memorizes vs generalizes. - Learning curves: train vs val loss / accuracy - Train ≫ val → overfit (regularize) - Train high AND val ≈ train → underfit (increase capacity) - Sweet spot: train low, gap < 10 % - Plot metric vs epoch to spot epoch where overfit starts

Phase 3 — Modeling & Optimization

Phase 3 deliverables

  • Best model registered in MLflow with seed + git SHA + dataset version
  • Learning curves saved as artifacts
  • HPO study saved (study.pkl) for replay

→ Deep deck: 03_model_training.html.

Phase 4 — Evaluation & Interpretation

Phase 4 — Evaluation & Interpretation

Classification metrics

Measure ranking quality. - Confusion matrix (TP / TN / FP / FN) - Validation curve, learning curve, threshold curve, calibration curve, lift curve - Accuracy (only on balanced classes) - Recall — find positives (safety-critical) - Precision — fewer false alarms (cost-critical) - F1 — balanced trade-off - ROC-AUC + Precision-Recall AUC

Phase 4 — Evaluation & Interpretation

Regression metrics

Measure error of prediction. - R² / Adjusted R² — explained variance - Feature importance, explained variance - MAE — robust to outliers - MSE / RMSE — penalizes large errors - MSLE — for exponential growth targets - Residual plot, Breusch-Pagan heteroscedasticity, Q-Q plot - Probability plot, prediction-error scatter - LassoCV alpha vs error sweep - MAE = f(epoch)

Phase 4 — Evaluation & Interpretation

Residual analysis (regression)

Verify statistical validity. - Residual plot must be random (no pattern) - Breusch-Pagan: detect heteroscedasticity (variance not constant) - Normality test: Kolmogorov-Smirnov, Q-Q plot

Phase 4 — Evaluation & Interpretation

Interpretation (XAI)

Explain the "why" behind predictions. - Feature importance (variable weights) - SHAP — global beeswarm + local force plot + dependence plots - LIME — local explanation by perturbation - TreeInterpreter — exact decomposition for trees - Anchor — high-precision IF-THEN rules - Counterfactual — what minimal change flips the prediction

Phase 4 — Evaluation & Interpretation

Threshold optimization

Tune sensitivity to business cost. - Calibration curve - Search threshold on F1 / Recall over validation set - Persist threshold in model bundle — inference reads it, never hard-code 0.5

Phase 4 — Evaluation & Interpretation

Phase 4 deliverables

  • Model card (one JSON sidecar) with all metrics + threshold + SHAP top-15
  • Confusion matrix + ROC + PR + calibration plots saved
  • Approval gate signed by domain expert

→ Deep deck: 04_evaluation_acceptance.html.

Phase 5 — Dimensionality Reduction & Clustering

Phase 5 — Dimensionality Reduction & Clustering

Reduction

Visualize or simplify data. - PCA — preserves global variance (linear). Scree plot, cumulative variance. - t-SNE / UMAP — local structure (non-linear, 2D / 3D viz) - PHATE — global + local trajectories - Curse of dimensionality: when n_features > n_samples / 10 - Visualization tools: 2D PC scatter, biplot, 3D Bokeh / Plotly

Phase 5 — Dimensionality Reduction & Clustering

Clustering

Find natural groups. - K-Means — centroid-based, requires k upfront - Hierarchical — dendrogram, cut at desired level - DBSCAN — density-based, handles noise (label = −1)

Phase 5 — Dimensionality Reduction & Clustering

Cluster validation

Validate the relevance of the groups. - Elbow on inertia — find right k - Silhouette coefficient — cohesion of clusters - Calinski-Harabasz, Davies-Bouldin indices - Combined dashboard: 4-metric chart + per-cluster silhouette + dendrogram + PCA 2D viz + cluster-mean bar chart

Phase 6 — Time Series & Deep Learning

Phase 6 — Time Series & Deep Learning

Time-series specifics

Sliding window + signal-processing features. - Window size W, step S — typical W=256, S=W/2 (50 % overlap) - FFT features per window: peak freq, peak magnitude, energy in N bands. Skip DC bin (index 0). - Statistical features per window: RMS, crest factor, kurtosis, skew, peak-to-peak (9 per axis × 3 axes = 27 features) - Stationarity check: ADF + KPSS tests - Architecture: 1D CNN, TCN, LSTM, Transformer (escalation order)

Phase 7 — Deployment & Monitoring & Cyber

Phase 7 — Deployment & Monitoring & Cyber

Save / persist

Persist the trained model. - pickle or joblib (dump / load) - Save model AND transformers (scaler, imputer) — ideally as a single Pipeline - Or use MLflow log_model for versioning + registry

Phase 7 — Deployment & Monitoring & Cyber

Production deployment

Make the model accessible. - API: Flask, FastAPI - Containerization: Docker, Kubernetes - ETL pipeline upstream

Phase 7 — Deployment & Monitoring & Cyber

Production monitoring

Watch the model in the wild. - MLflow tracking + GitHub Actions CI/CD - Rollback to stable version on regression - Drift detection (PSI per feature) + automatic retrain trigger - Alerting with thresholds, email, security logs - Latency p95 + throughput (req/s) of prediction calls - Volume + traffic of requests handled - Infra health: CPU, memory, storage

Phase 7 — Deployment & Monitoring & Cyber

Cyber security

Lock down the surface. - OAuth 2.0 for auth - Encryption: TLS in transit, encryption at rest (HTTPS) - Rate limiting - Strict input validation (avoid injection attacks) - Data freshness (avoid stale-input prediction) - Access monitoring + audit log

Phase 7 — Deployment & Monitoring & Cyber

Catalog the project ecosystem

Know your boundaries. - Identify all I/O of the data: source → store → API → consumer - Document the network around the project: deployment method, dependencies - Different feature exports (CSV, parquet, time-series store)

Phase 7 — Deployment & Monitoring & Cyber

Phase 7 deliverables

  • Deployed API with /health, /predict, /version routes
  • MLflow registered model + Staging tag
  • CI/CD pipeline green
  • Security review signed
  • Monitoring dashboard with SLO alerts wired

Closing — golden rules

Closing — golden rules
  • Always run a baseline. Beat it by ≥ 5 %, or stop and investigate.
  • Always log seeds, dataset version, git SHA, hyperparameters in MLflow.
  • Always compute BOTH train_loss AND test_loss. Symmetric overfit + underfit detection.
  • Never deploy without a rollback target. Register the previous champion.
  • Never deploy without monitoring + drift detection. Drift is silent until it's not.
  • Never trust correlation as causation. Use causal inference for "did X cause Y?" questions.
