🎵 Music Platform Dashboard — Onboarding Checklist

Follow each step in order. Estimated setup time: ~15 minutes.

1 — Prerequisites local machine

  1. Install Docker Desktop and enable WSL2 integration (Windows: Settings → Resources → WSL Integration).
  2. Install Python 3.11+ and confirm: python --version
  3. Clone or unzip the project into a working directory.
  4. Copy config template: cp config/config.example.yaml config/config.yaml
  5. Edit config/config.yaml — fill in database credentials and your Fernet key.
    Generate a key if needed: 
    python -c "from cryptography.fernet import Fernet; print(Fernet.generate_key().decode())"

2 — Start Docker (PostgreSQL + Airflow) infrastructure

  1. Open a terminal in the project root and run: 
    docker-compose up -d
  2. Wait ~30 seconds, then verify all containers are healthy: 
    docker-compose ps
    All services should show Up or healthy.
  3. Confirm Airflow UI is reachable: http://localhost:8080 (admin / admin)
  4. Confirm PostgreSQL is reachable on port 5433.
If a container fails, run docker-compose logs -f <service-name> to diagnose.

3 — Launch the Dashboard streamlit

  1. Install Python dependencies (first time only): 
    pip install -r requirements.txt
  2. Start the dashboard: 
    cd src/dashboard && streamlit run app.py
  3. Open http://localhost:8501 in your browser.
  4. Log in with the credentials defined in config/config.yaml → auth.credentials.
    If no auth section exists, the app starts in DEV MODE (auto-logged in as admin — a yellow banner will appear).

4 — Connect API Credentials credentials

  1. Navigate to 🔑 Credentials API in the sidebar.
  2. For each platform (Spotify, YouTube, Meta, SoundCloud, Instagram):
    • Click the platform tab.
    • Enter the required tokens / client IDs.
    • Click Test to verify the connection.
    • Click Save — credentials are Fernet-encrypted in the database.
  3. Confirm at least Spotify credentials are valid (required for ML scoring).
Credentials are per-artist. If you manage multiple artists, repeat this step after selecting each artist in the Admin panel.

5 — Trigger a DAG airflow

  1. Go to the Airflow UI: http://localhost:8080
  2. Find the DAG you want to run (e.g. spotify_api_daily).
  3. Toggle the DAG ON (blue switch on the left).
  4. Click the ▶ Trigger DAG button (top right of the DAG row).
  5. Watch the run turn green in the graph view — this confirms data is flowing into PostgreSQL.
Alternatively, use the 🚀 Lancer TOUTES les collectes button in the dashboard sidebar to trigger all DAGs at once.

6 — Upload a CSV import

  1. Navigate to 📂 Import CSV in the sidebar.
  2. Select the artist target (admin only) and the platform (S4A or Apple Music).
  3. Upload the CSV exported from Spotify for Artists or Apple Music for Artists.
  4. Review the Validation panel — all required columns should show ✅.
  5. Review the Preview (first 10 rows).
  6. Click ✅ Confirmer l'import to write to the database.

7 — Read the KPIs dashboard

🏠 HomeGlobal KPIs, source freshness, ROI summary.
🚀 Road to Algo (ML)J+28 stream trajectory + algorithm probability scores.
🎵 Spotify & S4APer-song stream history, audience trends.
📱 Meta AdsCampaign spend, CTR, CPR — linked to Spotify saves.
🎬 YouTubeSubscribers, views, top videos by type.
🎁 Data WrappedYear-over-year listener, stream and save evolution.
📄 Export PDFPrintable report per artist.
🏗️ Monitoring ETL(Admin) DAG health, last run, rows inserted.

8 — Troubleshooting Quick Reference ops

  • Dashboard cannot connect to DB → run docker-compose up -d and wait 30 s.
  • DAG fails immediately → check credentials via 🔑 Credentials API.
  • CSV import — column validation fails → confirm you selected the correct platform for the file.
  • ML scores missing → unpause ml_scoring_daily in Airflow UI and trigger it manually.
  • View shows no data → trigger the relevant DAG or import the corresponding CSV.
  • Rebuild after code changes: docker-compose build && docker-compose up -d
